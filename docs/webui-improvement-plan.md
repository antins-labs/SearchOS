# WebUI 优化开发计划

## 1. 背景与目标

SearchOS WebUI 已具备搜索编排、多轮追问、覆盖表、证据、工作区文件、模型与技能设置等核心能力。当前优化重点不再是增加装饰，而是提高以下四类体验：

1. **可靠反馈**：用户每次点击都能知道操作已接收、处理中、成功或失败。
2. **研究可操作性**：答案、表格和证据不仅能查看，还能筛选、修复、导出和复用。
3. **多轮可追溯性**：每轮研究的状态、证据和表格变化都可恢复、比较。
4. **跨尺寸可用性**：桌面、平板和手机都能完成核心工作流。

## 2. 实施原则

- 每次只完成一个编号条目，避免并行改动扩大回归范围。
- 每个条目都经过 TypeScript、定向 lint、生产构建和必要的浏览器验证。
- 保持现有 API、数据模型和用户工作区兼容；需要变更契约时先补测试。
- 优先复用现有设计 token、组件和后端能力，不引入第二套视觉体系。
- 错误不能静默吞掉；危险操作必须可确认或可撤销。
- 桌面端现有工作流不得因响应式改造退化。

## 3. 当前基线

已实测的主要问题：

- `390px` 视口下历史栏固定占用 `264px`，主内容仅剩约 `126px`。
- WebSocket 非正常断开时，运行中的任务可能被前端误判为完成。
- 历史多轮会话只有最后一轮恢复 `searchState`，前序回合无法查看 Coverage/Evidence。
- 历史加载、停止、重命名、删除等操作存在静默失败。
- 长答案和最终表格缺少复制、筛选、全屏和导出能力。
- Evidence UI 未展示数据模型中已有的 `support/conflict/refine` 关系。
- 表格拖拽创建、主键和外键编辑缺少完整校验与移动端布局。
- 设置页缺少 Provider、搜索后端和页面抓取后端的一键连通性测试。

## 4. 路线图

### 阶段 P0：可用性与可靠性

#### P0-01 响应式壳层与移动端工作流

**状态：已完成**

范围：

- 小屏历史栏改为遮罩侧滑抽屉。
- 小屏 Activity 改为全屏工作区。
- 设置弹窗改为全屏布局，设置分类支持横向滚动。
- 对话正文、底部输入框、建议项适配窄屏。
- 多表关系编辑器在窄屏下改为纵向布局。

验收标准：

- 在 `390x844` 下主内容宽度接近完整视口，不出现单字竖排。
- 历史栏可打开、关闭、选择会话；打开时有遮罩且不影响底层操作。
- Activity 可全屏打开并关闭，Coverage、Evidence、Files、Events 均可访问。
- 设置弹窗在手机上无横向溢出，分类导航可完整访问。
- `1365px` 桌面下继续保持三栏布局。

#### P0-02 统一异步操作反馈

**状态：已完成**

范围：

- 为历史加载、停止任务、重命名、删除和文件加载增加统一状态。
- 引入 `idle/loading/success/error` 操作模型和可复用反馈组件。
- 防止重复提交；删除支持撤销或失败回滚。
- 错误信息包含操作对象和可执行的下一步。

验收标准：

- 所有用户触发的网络操作都至少具有 loading 和 error 反馈。
- 请求失败时 UI 不保留错误的乐观状态。
- 连续快速点击不会发送重复请求。

实现说明：

- 历史列表、会话打开、重命名、删除、停止任务、工作区文件和文件内容均接入明确的加载、成功与失败状态。
- 新增统一 Toast 语义和可复用的 `AsyncFeedback` 组件，错误提示包含操作对象、失败原因与下一步。
- 历史操作与停止任务使用同步锁防止快速重复请求；删除改为服务端确认成功后再更新本地 UI，重命名失败时保留原状态。
- 搜索、历史和工作区请求增加请求头及响应体超时，文件轮询避免慢请求重叠。

#### P0-03 WebSocket 重连与运行状态修复

**状态：待开始**

范围：

- 增加 `reconnecting` 状态和指数退避重连。
- 断线后通过结果 API 判断 `running/completed/error`，不再二态推断。
- 重连后继续事件去重、计时和 Coverage 轮询。
- 明确显示“连接中断，正在恢复”。

验收标准：

- 运行中主动断开 WebSocket 不会显示为已完成。
- 网络恢复后可继续接收事件，历史事件不重复。
- 后端任务确实完成或失败后才进入终态。

#### P0-04 多轮历史状态恢复

**状态：待开始**

范围：

- 为每一轮保存或重建对应的 SearchState 快照。
- 前序回合 Activity 能查看当时的 Coverage 和 Evidence。
- 明确区分“当时状态”和“会话最新状态”。

验收标准：

- 三轮历史会话中任意一轮都能打开对应证据。
- 前序回合不会错误展示最后一轮新增的数据。
- 旧工作区缺少快照时显示兼容提示，而不是空白误导。

### 阶段 P1：结果工作台

#### P1-01 答案操作栏

**状态：待开始**

- 复制答案、导出 Markdown、折叠长答案、重新运行、基于本轮继续。
- 对复制和导出显示明确成功反馈。

#### P1-02 表格工作台基础能力

**状态：待开始**

- 行搜索、列筛选、排序、冻结表头与主键列、隐藏列、全屏查看。
- 保持多表与关系信息，避免把所有表强制拍平。

#### P1-03 CSV/XLSX/JSON 与研究包导出

**状态：待开始**

- 导出单表、多表和关系定义。
- 研究包包含答案、Coverage、Evidence、来源和运行元数据。
- 复用现有 CoverageMap 导出逻辑，新增 Web API 契约测试。

#### P1-04 建表与关系校验

**状态：待开始**

- 增强拖拽反馈：网格、尺寸、撤销、重做。
- 校验空实体名、重复列名、重复主键值、自关联和重复关系。
- 对 `N:N` 关系明确中间表语义。
- 支持粘贴 CSV/TSV 快速生成表结构。

#### P1-05 Activity 抽屉调整与状态保持

**状态：待开始**

- 桌面端支持拖拽调整宽度。
- 记住用户上次选择的 Tab 和宽度。
- Coverage 宽表优先获得可读空间。

#### P1-06 交互语义与键盘可用性

**状态：待开始**

- Pill、Tab、表格单元格补充 `aria-pressed/selected` 和键盘操作。
- 设置弹窗与证据弹窗增加焦点管理。
- 修复下拉菜单 Escape 同时关闭父弹窗的问题。
- 修正单复数、按钮可访问名称和低对比文本。

### 阶段 P2：SearchOS 特征能力

#### P2-01 定向补洞（Repair Cells）

**状态：待开始**

- 选择 missing、uncertain、hard cell 后发起定向补全。
- 只创建与所选单元格相关的 Frontier 任务。
- 展示修复前后状态和新增证据。

#### P2-02 证据冲突中心

**状态：待开始**

- 展示 `support/conflict/refine` 关系。
- 对冲突来源并排比较时间、摘录、质量与置信度。
- 支持用户确认采用来源或要求再次核验。

#### P2-03 研究版本与表格 Diff

**状态：待开始**

- 每轮保留 CoverageMap 快照。
- 展示新增、修改、删除、冲突解决的单元格。
- 支持从某一轮继续或复制为新会话。

#### P2-04 每次运行的 Skills 与来源控制

**状态：待开始**

- 在 Run overrides 中暴露后端已有的 per-run skill overrides。
- 支持可信域名、排除域名和只使用指定访问技能。
- 运行卡片展示本轮实际使用的 Skills。

#### P2-05 Provider 与后端连通性测试

**状态：待开始**

- Provider 测试模型调用、延迟和 Thinking 参数兼容性。
- 搜索后端测试查询能力。
- Browser backend 测试 URL 获取与代理配置。

#### P2-06 运行成本与资源面板

**状态：待开始**

- 展示 token、查询次数、耗时和模型分布。
- 在运行前根据 Effort 提供预算预估。
- 在运行后支持按 Agent 查看资源消耗。

## 6. 每项完成定义

一个条目只有同时满足以下条件才可标记完成：

- 功能实现并覆盖空态、加载态、成功态、失败态和禁用态。
- 相关 TypeScript 检查和定向 lint 无新增问题。
- 生产构建通过。
- 涉及视觉或交互时，完成桌面和移动端浏览器验证。
- 需要后端契约变更时，补充 API 测试。
- 更新本文档中的状态和必要的实现说明。
