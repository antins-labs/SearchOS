# Anti-patterns — constraint_satisfaction

## Index
- **一次性长查询** — 所有条件塞一个 query → 0 结果 (×1, 2026-04-18)
- **忽略隐含约束** — 忘了加时间/地点限定 (×1, 2026-04-18)
- **未记录来源** — 凭记忆复述无法回溯 (×1, 2026-04-18)
- **查询关键词截断** — 专有名词/标题截半失去精确匹配 (×2, 2026-04-18)
- **反复搜索陷入停滞** — 复杂约束单查询搜不到反复改词 (×1, 2026-04-19)
- **过早锚定候选** — 基于非独特属性锁定名字后补验证 (×1, 2026-04-19)

## Details

### 一次性长查询
**踩坑**: `"American painter sculptor 1960s six works Zeus mortal woman 17th century car"` 一次搜 → 0 结果
**原因**: 搜索引擎对长查询字面匹配极严，命中概率接近 0；且未利用"独特约束先行"的优势
**改用**: 先用最独特的 1-2 个约束搜候选列表（如 `"six works Zeus mortal woman 1960s"`），再用剩余条件逐项过滤
**观察**: 1 次  |  **最后**: 2026-04-18

### 忽略隐含约束
**踩坑**: 找到"符合身份"的候选却忽略"1930 年代"这类时间条件，最终答案不满足原始问题
**原因**: 拆条件时漏掉了连词/介词携带的隐含约束（时间段、地点、数量修饰）
**改用**: 条件拆解阶段显式枚举所有限定词并编号 C1…Cn；反向验证时对每个 Ci 独立确认
**观察**: 1 次  |  **最后**: 2026-04-18

### 未记录来源
**踩坑**: 凭记忆说"某百科写了"，却无法定位具体页面，用户要求溯源时答不出
**原因**: 验证过程中没有同步保存 URL，context 被后续搜索覆盖后信息即丢失
**改用**: 每验证一个 Ci 立即保存其权威来源 URL，形成 `C1: URL1, C2: URL2, …` 证据链
**观察**: 1 次  |  **最后**: 2026-04-18

### 查询关键词截断
**踩坑**: 查询时把专有名词/标题截断（"Stay wi" 代替 "Stay with the Fountain of Sorrow"；格式指令"determine when" vs 原文"determine the t"）
**原因**: 截断破坏精确匹配，搜索返回不相关页面；对约束本身也是违反（用户要 exact title 却没给）
**改用**: 专有名词、引文、格式指令原文照录；不确定时加引号限定 exact match
**观察**: 2 次  |  **最后**: 2026-04-18  |  **来源**: "DPO dataset mining"

### 反复搜索陷入停滞
**踩坑**: 面对"只产某种酒""距离范围"这类跨页面才能验证的约束，反复变换搜索词 `{keywords} distance` / `{keywords} only produces`，触发 NO_PROGRESS
**原因**: 单一搜索词难覆盖多页面综合的约束；搜预算快速耗尽而约束仍未核实
**改用**: 派发子代理专门核查，把搜索转结构化任务 `DISPATCH task="验证：1. [条件A] 2. [条件B]"`，利用独立 context 解决复杂逻辑
**观察**: 1 次  |  **最后**: 2026-04-19  |  **来源**: "Search trace mining"

### 过早锚定候选
**踩坑**: 基于部分匹配（如"music director"这种常见属性）就锁定某个候选名字，然后只围绕该候选补验证罕见约束
**原因**: 通用传记常忽略罕见属性（小众乐器、特定经历），导致"无法确认"的伪死局；如果锚点本身错了，所有验证白做
**改用**: 先识别最罕见的约束（如"同时会钢琴和小号"），用它产生候选列表；只有当独特约束都匹配后才锚定名字
**观察**: 1 次  |  **最后**: 2026-04-19  |  **来源**: "80 年代生音乐人 钢琴小号"
