# Anti-patterns — reverse_clue_search

## Index
- **从最显眼的泛词入手** — "知名电视剧""高分剧"候选一大堆 (×1, 2026-04-18)
- **一口气长查询 bundle all** — 多约束塞成一条 query → 0 结果 (×5, 2026-04-18)
- **跳过 intermediate 直接搜终答** — 不分解/zero-steps (×4, 2026-04-18)
- **过早锚定候选** — 约束未满足就钉人名 (×3, 2026-04-18)
- **混用关系类型** — "出演"/"祖籍"/"出生地" 不区分 (×1, 2026-04-18)
- **忽略高特异性线索入口** — 用泛线索而非 rarest signal (×2, 2026-04-19)

## Details

### 从最显眼的泛词入手
**踩坑**: 面对"某知名电视剧改编自小说…"直接搜"知名电视剧""高分剧""改编自小说"
**原因**: 这类词候选池极大，搜索引擎无法快速缩小范围；且往往命中讨论贴而非事实链
**改用**: 优先找"怪限制"（带出生地的剧名、特定乐器组合、特定年份的科举小试官姓）作为 entry，用 rarest clue 当钥匙
**观察**: 1 次  |  **最后**: 2026-04-18

### 一口气长查询 bundle all
**踩坑**: 把原始多约束问题（≥3-4 个独立属性）拼成一条 query 交给搜索引擎，如 "1980 年代北京音乐院校 小号 美国 工作室 空灵男歌手 2025 专辑"
**原因**: Keyword 太多触发 keyword dilution；搜索引擎对长 conjunctive query 处理不稳定；跨域属性组合往往无任何文档同时提及，recall 归零
**改用**: 拆成 ≥3 个独立子查询，每次只解一个中间变量。先选 sparsest/rarest 的约束做 entry（如"空灵男歌手 音乐总监"一个稀有组合），再用它产生的候选池用剩余约束过滤
**观察**: 5 次  |  **最后**: 2026-04-18  |  **来源**: "画家文学作品 / 小说家将军同月同日生 / 跨年晚会 / 音乐人多约束"

### 跳过 intermediate 直接搜终答
**踩坑**: 多跳链式问题直接 TASK_COMPLETE 0 步不搜；或直接搜最终目标没求中间变量（如 "capital of country where inventor of WWW was born" 一次搜完）
**原因**: 被条件链长度吓退；或不识别为 reverse_clue 任务；或试图让搜索引擎自己推导嵌套关系
**改用**: 任何多跳链式问题先拆 hop（inventor → Tim Berners-Lee → UK → London），每 hop 独立搜，拿到结果塞下一跳 query；泛题也先做一次初步搜索再看要不要反推
**观察**: 4 次  |  **最后**: 2026-04-18  |  **来源**: "DPO 多跳链 / Mercedes Sosa"

### 过早锚定候选
**踩坑**: 看到"顶流男明星"就 assume 是王鹤棣，派 3 个 agent 围着他查；或者 "Pieter Rijke 学生拿诺贝尔" 中被 Nobel 这个 high-profile 属性带偏，反复查 Lorentz/van der Waals 获奖年而非 Rijke 师生关系
**原因**: 著名次级属性（Nobel、顶流）hijack 搜索路径；wrong anchor 后所有下游验证变成 confirmation loop 而非 constraint-driven 收敛
**改用**: 约束全部独立满足前不钉名字；用最罕见的 constraint（花姓小试官、academic-supervision）当 primary query，只有 rare clues 都匹配后才 anchor 候选
**观察**: 3 次  |  **最后**: 2026-04-18  |  **来源**: "花姓主考 / 顶流男明星 / Pieter Rijke"

### 混用关系类型
**踩坑**: "出演"/"主演"/"男主演"/"女主演"/"祖籍"/"出生地" 被当作同义词处理
**原因**: 关系精度缺失——男主演 ≠ 任意出演；祖籍 ≠ 出生地；这些区分在反推题里往往就是唯一性的关键
**改用**: 拆题时显式标注每个关系的精度（男主 / 配角 / 父辈籍贯），query 里也用对应专有词
**观察**: 1 次  |  **最后**: 2026-04-18

### 忽略高特异性线索入口
**踩坑**: 题里有 "2012 年拍卖记录""2025 年春逝世""2014 翻唱 1979 老歌"这类 high-specificity 事件，却拿 "CAFA 杜塞尔多夫校友""母亲角色帮主角洗冤"这类 generic 描述当 entry
**原因**: 泛描述命中噪声且候选爆炸；高特异性事件（obituaries, auction records, specific dates）文档稀疏但精确，通常唯一锚定中间实体
**改用**: 扫题面找 1 个最稀有最具体的信号（具体日期、具体纪录、罕见组合），用它做 Step-1 query 锁定中间实体，再向其他约束扩展
**观察**: 2 次  |  **最后**: 2026-04-19  |  **来源**: "2012 拍卖纪录 / 2025 春逝世演员"
